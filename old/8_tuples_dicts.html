רשומות ומילונים#

רשומה tuple: “רשימה” בלתי ניתנת לשינוי#

  • tuple היא מבנה נתונים הדומה ל־list, אך הוא בלתי־ניתן לשינוי (immutable).

t = ("don't", "worry", "be", "happy") # definition

 t[0] # indexing

"don't"

t[-1] # backwords indexing

'happy'

t[1:3] 	# slicing

('worry', 'be')

my_list = [1, 2, 3]
my_tuple = (1, 2, 3)
my_tuple[0:2] 

(1, 2)

my_tuple[1] = 10

---------------------------------------------------------------------------
TypeError                                 Traceback (most recent call last)
Cell In[8], line 1
----> 1 my_tuple[1] = 10

TypeError: 'tuple' object does not support item assignment

אין אפשרות לקרוא לשום מתודה שראינו בlist שמשנה את הערך המקורי כמו append, remove וכו’

מילונים: dict: “רשימה” עם מפתחות במקום אינדקסים#

  • מבנה נתונים אסוציאטיבי

    • התאמה בין מפתח לערך (Key – Value)

    • גישה לערכים לפי מפתח (ולא לפי מיקום כמו ברשימות)

    • אין הבטחה לסדר של האיברים

    • מהיר מאוד!

  • אינטואיציה - דפי זהב:

    • אם יש לך שם, קל למצוא את מספר הטלפון המתאים

    • אם יש לך מספר טלפון, קשה להתאים לו שם

  • המפתחות צריכים להיות:

    • ייחודיים

    • בלתי ניתנים לשינוי (immutable)

תחביר:

{key1:val1, key2:val2,…}

דוגמא#

phonebook = {'Eric Cartman': '2021', 'Stan March': '5711', 'Kyle Broflovski': '2781'}
print(phonebook)

{'Eric Cartman': '2021', 'Stan March': '5711', 'Kyle Broflovski': '2781'}

  • הערה: באופן כללי, זוגות המפתח-ערך במילון לא בהכרח יוצגו באותו סדר שבו הוגדרו.
    מילונים לא מבטיחים סדר מסוים של מפתחות.

  • החל מ-Python 3.7, הסדר של המילון מובטח להיות לפי סדר ההכנסה,
    אך לא מומלץ להסתמך על כך משום שזה תלוי בגרסה/שפה.

  • גישה לפי מפתח:

phonebook['Eric Cartman']

'2021'

  • בדיקה אם מפתח קיים:

'Kenny McCormick' in phonebook

False

  • הוספה:

phonebook['Kenny McCormick'] = '1234'
print(phonebook)

{'Eric Cartman': '2021', 'Stan March': '5711', 'Kyle Broflovski': '2781', 'Kenny McCormick': '1234'}

מה קורה כאשר מוסיפים מפתח שכבר קיים?

print(phonebook)
phonebook['Kenny McCormick'] = '2222'
print(phonebook)

{'Eric Cartman': '2021', 'Stan March': '5711', 'Kyle Broflovski': '2781', 'Kenny McCormick': '1234'}
{'Eric Cartman': '2021', 'Stan March': '5711', 'Kyle Broflovski': '2781', 'Kenny McCormick': '2222'}

איך נוכל להוסיף עוד כניסה עבור Kenny McCormick בפנקס הטלפונים?

רעיון: להוסיף כתובת למפתח

phonebook= {['Kenny McCormick', 'Southpark']: '2222'}

---------------------------------------------------------------------------
TypeError                                 Traceback (most recent call last)
Cell In[14], line 1
----> 1 phonebook= {['Kenny McCormick', 'Southpark']: '2222'}

TypeError: unhashable type: 'list'

מה הבעיה?

  • מפתחות חייבים להיות בלתי ניתנים לשינוי (immutable)!

  • מה אפשר לעשות?

תיקון: השתמשו ב־tuples בתור מפתחות!

phonebook= {('Kenny McCormick', 'Southpark'): '2222'}
print(phonebook) 

{('Kenny McCormick', 'Southpark'): '2222'}

Dictionary methods#

פונקציה

תיאור

D = {}

יוצר מילון ריק

D[k] = value

קובע את הערך של D[k]

k in D

מחזיר True אם k הוא מפתח במילון, אחרת False

D[k]

מחזיר את הערך של המפתח k (יזרוק KeyError אם k לא קיים)

D.get(k, [d])

מחזיר את D[k] אם קיים, אחרת את d (ברירת מחדל: None)

D.keys()

מחזיר תצוגה של כל המפתחות במילון

D.values()

מחזיר תצוגה של כל הערכים במילון

D.items()

מחזיר תצוגה של זוגות (מפתח, ערך) כ־tuples

D.pop(k, [d])

מסיר את המפתח k ומחזיר את ערכו; אם לא נמצא מחזיר d או שגיאה

D.copy()

יוצר העתק רדוד של המילון (המילון בלבד מועתק, לא האובייקטים שבתוכו)

D.update(other)

מוסיף את כל הפריטים מהמילון other לתוך D

  • ארגומנט שמסומן כ־[d] הוא אופציונלי

    • אם לא מציינים אותו, ייעשה שימוש בערך ברירת המחדל

  • מידע נוסף על מתודות של מילון נמצא כאן

תצוגות מילון (Dictionary Views)#

  • המתודות של מילון keys(), values(), items() מחזירות תצוגה איטרבילית דינמית.

  • החל מ-Python 3.7, סדר התצוגה של המילון נשמר לפי סדר ההכנסה, אבל לא כדאי להסתמך על כך.

  • ניתן לאחזר תצוגה בלולאה, לבדוק שייכות עם in, והיא משתנה דינמית כאשר המילון משתנה.

  • התצוגה משתנה כאשר המילון משתנה

    • לא לשנות את המילון תוך כדי איטרציה על אחת מהתצוגות שלו

d={'A':65, 'B':66, 'C':67}
print(d)
print(d.keys())
print(d.values())
print(d.items())
a=list(d.items())

{'A': 65, 'B': 66, 'C': 67}
dict_keys(['A', 'B', 'C'])
dict_values([65, 66, 67])
dict_items([('A', 65), ('B', 66), ('C', 67)])

a=d.keys()
print(a)
d['D']=68
d['E']=69
print(a)
print('E' in a)

dict_keys(['A', 'B', 'C'])
dict_keys(['A', 'B', 'C', 'D', 'E'])
True

  • מטרה: לספור את התדירות של כל אות במחרוזת

def get_char_count(text):
    char_count = {}
    for char in text:
        if not char in char_count:
            char_count[char] = 1
        else: 
            char_count[char] += 1
    return char_count

get_char_count(text)

מונה תדירות תווים משופר#

  • שימוש ב-get():

def get_char_count(text):
    char_count={}
    for char in text:
        char_count[char] = char_count.get(char, 0) + 1
    return char_count

text = 'jerry'
get_char_count(text)

{'j': 1, 'e': 1, 'r': 2, 'y': 1}

מונה תדירות תווים: הדפסת תוצאה ממוינת#

באיזה סדר נדפיס את התווים?

אפשרות ראשונה - לפי סדר אלפביתי: תחילה את מספר המופעים של a, לאחר מכן את מספר המופעים של b וכו’

def print_by_keys_order(d):
    sorted_chars = sorted(d)
    for char in sorted_chars:  
        print(char, ':', d[char])

text = 'supercalifragilisticexpialidocious'
cc = get_char_count(text)
print_by_keys_order(cc)

  • אפשרות שניה - לפי תדירות: להתחיל מהאות שמופיעה הכי הרבה פעמים ולסיים בזו שמופיעה הכי מעט

def print_by_keys_order(d):
    sorted_chars = sorted(d, key=d.get, reverse=True)
    for char in sorted_chars: 
        print(char, ':', d[char])
